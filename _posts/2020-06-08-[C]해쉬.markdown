## 해쉬란?

해쉬는 임의의 크기를 가진 데이터를 고정된 크기로 변환시키는데 사용되는 기법

### 1. Direct Addressing Table

key - value 쌍의 데이터를 배열에 저장하기 위해 key 값을 배열의 인덱스로 사용하는 방법이다.

즉, 데이터의 값이 100 이라면 배열의 인덱스가 100인 위치에 키 값을 저장하고 포인터로 가리킨다.

똑같은 키가 존재하지 않는다고 가정하면, 삽입 시 각 키마다 자신의 공간이 존재하므로 그 위치에다 저장을 하면 된다.

따라서 탐색시에는 해당 키의 위치를 바로 참조하면 되기에 탐색, 저장, 삭제, 갱신 모두 O(1) 로 매우 빠른 속도로 처리가 가능하다.

하지만, key 값이 최대 크기만큼 배열이 할당되기 때문에 공간을 효율적으로 사용할 수 없다는 단점이 있다.

### 2. Open Addressing

Direct Addressing Table 과 달리, 모든 데이터를 (Key 값 + d) 의 인덱스에 저장하는 방식이다.

포인터를 사용하지 않는다.

#### 충돌 문제

포인터를 사용하지 않기 때문에 해쉬 테이블의 Channing 방법을 사용할 수는 없다.

##### Liner probing

key 값으로 인덱스에 접근하여 데이터를 저장할 때, 충돌이 날 경우 충돌이 일어나지 않을 때까지 다음 인덱스를 탐색하며 빈 공간에 데이터를 저장하는 방식이다.

구현은 용이하지만 primary clustering 이라는 문제점이 발생하는데, 이는 충돌이 날수록 뒷 슬롯에 데이터를 넣게 되어 데이터들이 특정 위치에만 밀집하게 되는 현상을 말한다.(탐색 시간이 늦어질 수 있음)

##### Quadratic probing

Liner probing 의 primary clustering 을 방지하기 위해 해쉬 함수를 2차식의 형태로 구현하여, 최대한 분포를 넓히는 방법이다. (충돌이 나더라도 군집이 모이는 현상을 최소화 하고자 함)

하지만 처음 시작시 해쉬 값이 동일한 값이 계속 들어올 경우, 해당 인덱스 주변으로 군집이 생기게 되며 primary clustering 문제가 발생하게 된다.

##### Double hashing

Quadratic probing 의 secondary clustering 을 해결하기 위해 사용하는 방법으로 해쉬 함수를 2개로 구성하는 방식이다.

h1(k) = k mod m
h2(k) = k mod m2
h(k, i) = (h1(k) + i*h2(k)) mod m

### 3. Hash Table

key - value 쌍에서 key 값을 테이블에 저장할 때, key 값을 `해쉬 함수로`  계산한다.(해쉬 함수는 입력으로 key 를 받아 0부터 배열의 크기 - 1 사이의 값을 출력한다.)

k 값이 있을 때, h(k) 라는 해쉬값을 얻게 되며 이 해쉬 값으로 배열의 인덱스로 사용된다.

Driect Addressing Table 와 달리 key 값의 크기에 테이블의 크기가 결정되지 않고, h(k)만큼 공간이 저장되기 때문에 비교적 공간을 더 효율적으로 사용할 수 있다.

#### 충돌 문제

해쉬 테이블은 해쉬함수의 결과갑승로 동일한 해쉬값이 생겨 인덱스가 충돌되는 경우가 존재한다.

(이를 방지하기 위해 Direct Addressing Table 의 경우 key 값이 모두 다르다)

해쉬 함수를 좋게 짜더라도 충돌을 '완전히' 방지한다는 것을 보장하기 어렵기 때문에, 충돌을 최소한으로 허용하는 방법이 널리 알려져 있다.

###### channing 방법 - 충돌을 허용하되 최소화하는 방법

해쉬값이 동일하면 해쉬값에 대응되는 인덱스에 위치해 있는 데이터들을 연결리스트로 저장하는 방식이다.

해쉬 과정을 제외하고, 모든 탐색, 삽입, 삭제 과정은 연결리스트와 유사한 방식으로 진행된다.

삽입은 해쉬값을 이용해 저장하면 되기에 상수시간이 발생하고 O(1), 삭제는 연결리스트의 삭제와 동일한 상수시간에 발생하고 O(1), 탐색은 연결리스트의 길이만큼 발생하게 된다.(On)

###### 적재율

해쉬 테이블은 O(a+1) 의 시간 복잡도를 가진다. 

a 는 적재율이며 현재 저장된 key 값의 개수 (K) 를 전체 테이블의 개수 (N) 으로 나눈 값을 의미한다. ( a = K / N )

즉, 저장된 데이터가 많으면 많을수록 충돌 확률이 높아져 연결 리스트 탐색 확률도 증가하며, 적을수록 리스트 탐색 확률이 적어진다.

###### Simple uniform Hash - 충돌을 최소화 하는 방법

충돌을 최소화 하는 해쉬 함수를 만드는 조건은 다음과 같다.

1. 계산된 해쉬값들을 0부터 배열의 크기 - 1 사이의 범위로 동일한 확률을 골고루 출력할 것

2. 각각의 해쉬값들은 서로 연관성을 가지지 않고 독립적으로 생성할 것

##### Division method

대표적인 해쉬 함수로 division method 가 있는데, modular 연산 방법을 이용한 방법이다. 특정 key 를 어떤 수로 나눈 나머지를 해쉬 값으로 사용한다.

`k mod m` 으로 m=100 이면 0부터 99까지의 범위를 가지는 방식이다. 이 때 m 값이 해쉬 테이블의 성능을 크게 좌우하는데, m 의 크기는 보통 key 의 수의 3배가 적당하다.(적재율이 30% 쯤까지 충돌이 거의 일어나지 않는다고 한다.)
